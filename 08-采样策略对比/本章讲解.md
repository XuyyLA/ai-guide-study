## 本章知识

### 1. 解码策略总览

```
解码策略
├── 确定性
│   ├── 贪心搜索（每步选概率最高的 1 个 token）
│   └── Beam Search（保留 k 个最优候选）
└── 随机性
    ├── Top-K 采样（只从概率最高的 K 个 token 中采样）
    ├── Top-P 核采样（从累积概率达到 P 的最小集合中采样）
    └── 组合：Top-K + Top-P + Temperature
```

### 2. Top-K 采样

每步只从概率最高的 **K** 个 token 中按概率随机采样，其余 token 概率置零后重新归一化。

- K=1：退化为贪心搜索
- K=50：常用值，平衡多样性和质量
- K=词表大小：等价于普通采样

**问题**：K 是固定的，无法适应分布变化。有时第 K 个 token 概率很高（不该截断），有时第 5 个就很低（应该截断）。

### 3. Top-P（核采样 / Nucleus Sampling）

选择概率从高到低累积达到 **P** 的**最小 token 集合**，然后在该集合内重新归一化采样。

- P=0.1：只选概率最高的几个 token，接近贪心
- P=0.9：常用值，覆盖 90% 概率质量
- P=1.0：不截断，等价于普通采样

**优势**：自适应候选数量。分布集中时选少量 token，分布平坦时选更多。

### 4. Temperature

Temperature 缩放 logits 后再 softmax：

```
probs = softmax(logits / temperature)
```

| Temperature | 效果 | 适用场景 |
|-------------|------|---------|
| → 0 | 概率分布极尖锐，趋向贪心 | 事实性任务（翻译、摘要） |
| 0.5-0.7 | 稍有随机性 | 代码生成、问答 |
| 1.0 | 原始分布 | 默认值 |
| 1.0-1.5 | 更随机 | 创意写作、故事 |
| → ∞ | 均匀分布 | 探索性任务 |

### 5. 实践建议

| 任务类型 | temperature | top_k | top_p |
|---------|-------------|-------|-------|
| 翻译/摘要 | 0.3 | — | — |
| 代码生成 | 0.2 | — | 0.95 |
| 对话/问答 | 0.7 | 50 | 0.95 |
| 创意写作 | 1.0-1.5 | 50 | 0.95 |

### 关键参数

```python
model.generate(
    do_sample=True,        # 启用随机采样
    temperature=1.0,       # 温度（控制随机性）
    top_k=50,              # Top-K（0 = 禁用）
    top_p=0.95,            # Top-P（1.0 = 禁用）
    num_beams=1,           # 采样模式下束宽必须为 1
)
```
