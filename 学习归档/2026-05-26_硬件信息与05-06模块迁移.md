### 06-微调LLM写唐诗（对应 Guide 08）

从 `Demos/06. 尝试微调 LLM：让它会写唐诗.ipynb` 迁移。

| 文件 | 说明 |
|------|------|
| `01微调前推理.py` | 加载 Qwen2.5-7B-Instruct，测试微调前写唐诗能力 |
| `02微调训练.py` | LoRA 微调训练（4-bit 量化 + SFTTrainer + completion_mask） |
| `03微调后推理.py` | 加载 LoRA checkpoint，测试微调后效果并对比 |
| `GenAI-Hw5/` | 唐诗数据集（训练 5001 条，测试 15 条，已转简体中文） |

**模型变更**：原教程使用 Breeze-7B（繁体中文），改为 Qwen2.5-7B-Instruct（简体中文）。

**格式变更**：
- Prompt 从 Llama `[INST]` 格式 → Qwen ChatML 格式（`apply_chat_template`）
- 输出解码从手动 `split("[/INST]")` → `decode(input_ids_len:, skip_special_tokens=True)`
- 设备选择从硬编码 `.cuda()` → `.to(device)` 自动适配

**数据集变更**：使用 opencc 将 GenAI-Hw5 中所有繁体中文转为简体。

## 知识架构

### 模型部署（05 模块）

```
本地模型加载 → 推理 → 封装为 API 服务
                     ├── FastAPI（高性能，自动文档，uvicorn 启动）
                     └── Flask（轻量，直接运行）
```

- **AutoModelForCausalLM**：因果语言模型，用于文本生成
- **model.generate() 参数**：max_length, num_beams, no_repeat_ngram_size, early_stopping
- **FastAPI vs Flask**：FastAPI 性能更好且有 Swagger UI，Flask 更简单

### LoRA 微调（06 模块）

```
预训练模型 → 4-bit 量化加载 → prepare_model_for_kbit_training
    → LoRA 注入（get_peft_model）→ SFTTrainer 训练 → 保存 checkpoint
    → 加载 checkpoint（PeftModel.from_pretrained）→ 推理对比
```

- **BitsAndBytesConfig**：4-bit NF4 量化，节省显存（7B 模型约 5.2G）
- **LoRA**：低秩适配，只训练少量参数（r=8, alpha=16, dropout=0.05）
- **SFTTrainer**：TRL 库的有监督微调训练器
- **completion_mask**：prompt 部分标记 0（不计算 loss），output 部分标记 1（计算 loss）
- **DataCollatorForLanguageModeling**：动态 padding + completion_only_loss
- **ChatML 格式**：Qwen 的对话模板，通过 `apply_chat_template` 自动构建
