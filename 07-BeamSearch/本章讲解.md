## 本章知识

### 1. Beam Search 原理

贪心搜索每步只选概率最高的 1 个 token，容易陷入局部最优。Beam Search 每步保留 **束宽 k** 个最优候选序列，扩大搜索空间。

**算法步骤**：
1. 初始化：从起始 token 开始，束宽 k
2. 扩展：对每个候选序列，生成所有可能的下一个 token
3. 选择：从所有扩展候选中选得分最高的 k 个
4. 终止：所有候选生成 `<eos>` 或达到最大长度
5. 输出：从完成序列中选得分最高的

### 2. 得分计算

使用**对数概率**（避免连乘导致数值下溢）：

```
score = log(P(t1)) + log(P(t2|t1)) + log(P(t3|t1,t2)) + ...
```

对数概率越大（越接近 0），序列概率越高。

### 3. `<eos>` 处理

当某个候选序列生成 `<eos>` 时：
- 将其移入"已完成"列表，不再扩展
- 其余候选继续搜索
- `early_stopping=True`：所有候选都完成时提前停止

### 4. 束宽对生成的影响

| 束宽 | 特点 |
|------|------|
| 1 | 等同于贪心搜索，速度快但可能重复/不连贯 |
| 3-5 | 平衡质量与速度，输出更连贯 |
| 更大 | 搜索更全面，但计算成本高，可能过度保守 |
