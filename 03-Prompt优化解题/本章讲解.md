## 本章知识

### 1. Chain of Thought (CoT)

通过 Prompt 引导模型"分步思考"，将推理过程显式写出，再从推理中提取最终答案。相比直接要答案，CoT 显著提升数学/逻辑题准确率。

### 2. 双智能体协作（Solver + Verifier）

```
用户问题 → Solver 求解 → 提取答案 → Verifier 验证 → 判断正误
                ↑                                    |
                └──────── 如果错误，重新求解 ←────────┘
```

- **Solver**：解题智能体，输出推理过程和答案
- **Verifier**：验证智能体，检查答案是否正确
- 多轮迭代直到答案正确或达到最大轮次

### 3. 答案提取

用正则表达式从模型输出中提取数值答案，与标准答案比对。

### 4. 评估机制

对每道题进行多次试验（默认 3 次），投票确定最终结果，统计准确率。

## 关键 Prompt 设计

Solver 的 system prompt 示例：
```
你是一个数学问题求解助手。请一步步思考，给出详细的推理过程，
最后用"答案是：X"的格式给出最终答案。
```

Verifier 的 system prompt 示例：
```
你是一个答案验证助手。请检查以下解答是否正确，
如果正确回复"验证通过"，否则指出错误。
```
