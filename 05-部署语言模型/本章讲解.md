### 1. 模型部署流程

```
加载模型到内存 → 接收 HTTP 请求 → 调用 model.generate() → 返回 JSON 响应
```

### 2. FastAPI vs Flask

| 特性 | FastAPI | Flask |
|------|---------|-------|
| 性能 | 高（ASGI 异步） | 中（WSGI 同步） |
| 自动文档 | Swagger UI (`/docs`) | 无 |
| 请求校验 | Pydantic 模型 | 手动取 `request.json` |
| 启动方式 | `uvicorn 文件名:app` | `python 文件名.py` |
| 学习曲线 | 稍陡 | 简单 |

## 代码讲解

| 文件 | 说明 |
|------|------|
| `01模型加载与推理.py` | 加载 distilgpt2，设置设备（GPU/MPS/CPU），`model.generate()` 生成文本 |
| `02FastAPI部署.py` | FastAPI 封装：`POST /generate` 接收 `{"prompt": "..."}` 返回生成结果 |
| `03Flask部署.py` | Flask 封装：同上接口，更轻量的实现 |

## model.generate() 关键参数

| 参数 | 说明 |
|------|------|
| `max_length` | 生成文本最大长度 |
| `num_beams` | Beam Search 束宽（越大越稳定但越慢） |
| `no_repeat_ngram_size` | 防止重复 n-gram |
| `early_stopping` | 所有候选生成 `<eos>` 时提前停止 |
